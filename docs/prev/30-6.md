- **File Upload:** Multer (disk storage runtime, memory storage test)
- **Email:** Nodemailer (Gmail SMTP, fallback log OTP khi thiếu SMTP config)

### 2.2. Triển Khai Async Extraction & OCR (Phần 1, 2, 3)

Triển khai hệ thống xử lý trích xuất văn bản bất đồng bộ dựa trên kế hoạch `async-extraction-ocr.md`.

- **Phase 1 (Database & Config)**:
  - Tạo model `DocumentExtractionJob.schema.ts`.
  - Thêm collection `documentExtractionJobs` vào `database.service.ts` kèm index (`status`, `lockedAt`, `solutionId`).
  - Định nghĩa config (Max attempts, lock TTL, poll interval) trong `base.ts`.
- **Phase 2 (Async Upload Separation)**:
  - Tách hàm `extractText` khỏi luồng `uploadDocument` trong `document.service.ts`. API hiện phản hồi ngay lập tức.
  - Khởi tạo tài liệu với trạng thái `pending`.
  - Enqueue job vào DB ngay khi lưu tài liệu, kèm cơ chế rollback nếu fail. Cập nhật `documents.test.ts` để pass behavior mới.
- **Phase 3 (Background Worker)**:
  - Bổ sung `documentExtractionJob.service.ts` với hàm `claimNextJob` (sử dụng atomic `findOneAndUpdate`) và các hàm cập nhật trạng thái (`completeJob`, `skipJob`, `failJob`).
  - Viết `documentExtraction.worker.ts` với vòng lặp polling liên tục, tự động ánh xạ tệp từ local/memory sang cấu trúc `Multer.File` giả lập.
  - Hỗ trợ graceful shutdown (SIGINT/SIGTERM) và cập nhật enum `ActivityAction` với giá trị `extractSkipped`.
- **Code Refactoring**:
  - Tách kiểu `RequestContext` (bị lặp ở `document.service` và `sharing.service`) ra file dùng chung `src/models/request/common.request.ts`.
- **Kết quả Test**: Unit tests (23/23) và Integration tests (30/30) đều pass hoàn toàn.

## 3. Cảnh Báo Kiến Trúc & Sai Lệch (Cần Lưu Ý Khi Code)

Các agent sau cần nắm rõ các sai lệch giữa tài liệu kiến trúc cũ và mã nguồn thực tế:

1. **Mongoose vs Native**: Tài liệu cũ ghi Mongoose, nhưng code **dùng mongodb native driver**.
2. **Response Envelope**: Trả về là `{ message, data }`, **không có trường `success`**.
3. **Verify Email Route**: API spec cũ ghi `GET`, code thực tế dùng `POST /account/verify-email` với body `{ email, otp }`.
4. **`defautHandler`**: Typo này ở global error handler là convention đang chạy trong code, không phải bug.
5. **Express-Validator**: Hàm `isEmail` chạy trước `trim`. Email truyền lên không được có dấu cách dư thừa ở đầu/cuối. Service sẽ tự normalize lại qua `email.toLowerCase().trim()`.
6. **Folders Collection**: Bị sót trong tài liệu `database-entities.md` cũ nhưng runtime thực tế đang dùng.
